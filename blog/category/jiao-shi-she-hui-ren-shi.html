建立網誌文章步驟

Code4future | Tue 07 November 2017

Code4future 的網誌目前採靜態 html 格式, 並且寄存在 https://github.com/code4futuredotcom/code4futuredotcom.github.io, 因此建立網誌文章的步驟, 就是編輯位於 Leo Editor 專案節點中的 Markdown 文章, 然後轉為近端格式後, 在近端瀏覽器中檢查格式是否正確後, 再轉為遠端格式後, 以 Git 指令, 新增、提交並推送到 Github 倉儲.

Leo Editor 簡介

Code4future | Tue 07 November 2017

Code4future 的網誌是利用 Leo Editor 編輯管理, 位於 content 目錄中的 Markdown 檔案是網誌的主要內容, 以 Leo Editor 的節點管理每一則 .md 檔案, 然後再將 Pelican 轉檔用的指令存成 Leo Editor 的 button, 在正常情況下, 只要利用滑鼠點到該 button 所在的路徑後, 再點擊命令執行轉檔.

Pelican 靜態網頁產生器

Code4future | Tue 07 November 2017

Pelican 是一種靜態網頁產生程式, 以 Python3 編寫, 可以將作者所編輯的 reStructuredText 或 Markdown 檔案, 配合網站 Template 及設定檔案, 轉換為一組網誌 html 超文件, 只需要 WWW 伺服器就可以在網站上呈現, 目前的 Github Pages 服務應該最適合靜態網誌的寄存.
